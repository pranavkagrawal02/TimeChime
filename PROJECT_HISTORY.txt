Current Data Setup
------------------
- For now the app uses db.json as the internal database.
- SQL Server support has now been added for local SSMS / SQLEXPRESS use.
- If SQL env vars are not set, the app still falls back to `db.json`.
- `server.js` now supports loading a local `.env` file automatically.
- Current SQL path expects:
  - server: `localhost`
  - database: `ScheduleTracker`
  - SQL login: `scheduleapp`
  - SQL password: `Schedule@123` (or the user-chosen replacement)
- Current blocker found on this machine:
  - the app now reaches SQL Server successfully with `scheduleapp`
  - the first `/api/bootstrap` call fails with: `CREATE TABLE permission denied in database 'ScheduleTracker'`
  - `scheduleapp` needs `db_ddladmin`, or `dbo.schedules` must be created manually by an admin user
